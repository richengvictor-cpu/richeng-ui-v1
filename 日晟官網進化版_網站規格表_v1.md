# 日晟官網進化版｜網站規格表 v1（最小可執行版）

## 1. 專案目標
- 建立一個清楚、可信、易理解的官網，讓訪客快速知道日晟是誰、提供什麼服務、如何聯絡。
- 以「提升詢問量」為第一優先，而不是先追求功能完整度。
- 先完成最小可執行版本 v1，作為後續 v1.1 / v2 的審定基礎。
- 網站重點放在：
  - 快速傳達服務內容
  - 降低聯絡門檻
  - 提升表單送出與電話 / LINE 轉換

## 2. 目標客群
- 需要了解日晟服務內容的潛在客戶
- 有明確需求、正在找合作對象或供應商的企業 / 個人
- 想先快速確認服務範圍、案例感受與聯絡方式的訪客
- 主要行為目標：
  - 先看懂服務
  - 先建立信任
  - 立即聯絡或留下需求

## 3. 網站 Sitemap
- 首頁
- 服務頁
- 聯絡我們
- 隱私權政策（必要頁，供表單與頁尾連結使用）

## 4. 首頁主要區塊
- Hero 首屏
  - 品牌主標
  - 一句服務價值說明
  - 主要 CTA
- 服務總覽
  - 以簡短方式列出主要服務項目
  - 引導進入服務頁
- 我們的優勢
  - 3 到 5 點核心優勢
  - 以可信度與差異化為主
- 流程簡述
  - 讓訪客知道合作大致怎麼進行
- 服務對象 / 適合情境
  - 說明哪些客戶適合找日晟
- 需求引導區
  - 鼓勵訪客直接聯絡
- 頁尾資訊
  - 聯絡方式
  - 基本連結
  - 隱私權政策

## 5. 服務頁主要區塊
- 服務標題與摘要
  - 清楚說明該服務是做什麼
- 服務內容說明
  - 列出實際提供的項目
- 適合對象
  - 讓訪客判斷是否符合自身需求
- 作業流程
  - 從諮詢到交付的基本步驟
- 常見疑問
  - 回答最常被問到的 3 到 5 個問題
- 轉換 CTA
  - 引導聯絡或填表

## 6. 聯絡表單需求
- 表單目的：讓訪客快速留下需求，方便後續聯繫
- 必填欄位：
  - 姓名
  - 聯絡電話或 Email 擇一
  - 需求說明
- 選填欄位：
  - 公司名稱
  - 服務類別
  - 預算區間
  - 預計時程
- 必要機制：
  - 勾選同意隱私權政策
  - 提交成功提示
  - 送出後顯示後續聯繫說明

## 7. CTA 規劃
- 首頁首屏 CTA
  - 立即洽詢
  - 查看服務
- 首頁中段 CTA
  - 了解合作流程
  - 直接留下需求
- 服務頁 CTA
  - 立即聯絡
  - 填寫需求表單
- 頁尾 CTA
  - 聯絡我們
  - 加入 LINE / 撥打電話（如日後確認要放置）

## 8. v1 範圍與非 v1 範圍

### v1 範圍
- 官網基本頁面架構
- 首頁內容骨架
- 服務頁內容骨架
- 聯絡表單基本功能
- CTA 引導設計規劃
- 隱私權政策必要頁

### 非 v1 範圍
- UI 設計稿與視覺提案
- 前端切版與程式開發
- 會員系統
- 金流功能
- AI 聊天機器人
- 複雜後台
- 多語系
- 登入 / 權限管理
- 進階數據儀表板
- v2 / v3 功能擴充
- 其他未經審定的新功能

- 是否需要案例頁或先保留到後續版本
